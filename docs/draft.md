  二、深度缺口分析（按交易逻辑排序）

  🔴 缺口1：连板梯队 + 情绪量化（最核心）

  短线交易者的灵魂问题：

  ▎ "今天市场情绪好不好？连板梯队健不健康？有没有赚钱效应？"

  现有接口的问题：
  - get_hot_stocks 只给当日涨停列表
  - 没有连板梯队统计（几板、最高板、断板）
  - 没有情绪量化指标（封板率、昨日涨停今日表现）

  应该扩展的能力：

  1. 连板梯队接口
     - 最高连板是几板？（高度标）
     - 5板以上有几只？3板有几只？首板有几只？
     - 梯队是否健康（高标断没断？低位有没有接力？）

  2. 情绪量化接口
     - 今日涨停家数 / 跌停家数
     - 封板成功率（炸了多少？）
     - 昨日涨停今日平均涨幅（赚钱效应核心指标）
     - 昨日涨停今日晋级率（2板以上的比例）
     - 连板股平均溢价

  为什么是核心：

  ▎ 情绪好 → 做高位接力；情绪差 → 空仓或只做首板。这是短线交易者的第一决策依据。

  ---
  🔴 缺口2：题材周期评估

  短线交易者的第二问题：

  ▎ "这个题材炒了几天了？还有没有空间？还是该撤了？"

  现有接口的问题：
  - get_concept_blocks 只给个股所属概念
  - 没有题材启动时间
  - 没有题材内涨停家数变化（从发酵到高潮到退潮）
  - 没有题材持续性评估

  应该扩展的能力：

  1. 题材周期接口
     - 概念板块启动日期（首次涨停潮）
     - 概念板块内涨停家数趋势（5→12→8→3？）
     - 概念板块内龙头股表现（还封着还是炸了？）
     - 题材轮动信号（A题材退潮，B题材启动）

  2. 热点追踪接口
     - 今日涨停原因归因（哪个题材贡献最多涨停）
     - 近3日涨停原因分布（题材持续性判断）
     - 新题材识别（首次出现的涨停原因）

  为什么是核心：

  ▎ 题材炒作有周期（启动→发酵→高潮→退潮→冰点），交易者需要知道当前处于哪个阶段，才能决定仓位和策略。

  ---
  🔴 缺口3：盘中异动 + 实时情绪监控

  短线交易者的盘中问题：

  ▎ "现在市场怎么样？要不要动手？什么时候动手？"

  现有接口的问题：
  - 有 get_stock_data 实时行情
  - 但没有全市场情绪聚合
  - 没有炸板实时监控
  - 没有板块资金流动态

  应该扩展的能力：

  1. 实时情绪监控接口
     - 当前涨停家数 / 炸板家数（动态更新）
     - 封板成功率实时变化
     - 市场高度标是否还封着
     - 昨日涨停股今日表现分布

  2. 炸板预警接口
     - 哪些票炸板了？炸板原因是什么？
     - 炸板后是否有资金回封？
     - 连板股炸板风险提示

  为什么是核心：

  ▎ 情绪拐点往往在盘中出现（比如10:30前封板率骤降），没有实时监控就无法捕捉。

  ---
  🟡 缺口4：卡位/补涨/龙头识别

  短线交易者的博弈问题：

  ▎ "这只票是真龙头还是假龙头？有没有卡位的？补涨龙是谁？"

  现有接口的问题：
  - 龙虎榜只给当日席位
  - 没有连板过程中的席位跟踪
  - 没有同题材内竞争关系分析

  应该扩展的能力：

  1. 龙头识别接口
     - 同题材内谁先涨停（首板时间）
     - 谁连板最高（高度标）
     - 封单量/流通盘比（封单强度）
     - 谁在卡位（同板数、同题材、后涨停但更强势）

  2. 席位风格画像
     - 某游资历史操作风格（打首板？做接力？）
     - 某游资胜率
     - 某游资近期活跃度

  为什么重要：

  ▎ 龙头战法的核心是识别真龙头、回避假龙头。现有龙虎榜数据可以做更多挖掘。

  ---
  🟡 缺口5：首板挖掘 + 二板预期

  短线交易者的核心战法：

  ▎ "今天首板的票，明天有没有二板预期？"

  现有接口的问题：
  - get_hot_stocks 给了涨停列表
  - 但没有首板票筛选
  - 没有二板预期评估（封单强度、题材热度、量价配合）

  应该扩展的能力：

  1. 首板筛选接口
     - 今日首板票列表
     - 首板封单强度排名
     - 首板所属题材热度
     - 首板量价配合度（缩量板 vs 放量板）

  2. 二板预期评估
     - 基于封单强度、题材持续性、量价关系
     - 给出二板概率评分

  为什么重要：

  ▎ 首板打板是短线核心战法之一，但需要从大量首板票中筛选出有二板预期的标的。

  ---
  🟡 缺口6：高标股状态监控

  短线交易者关注：

  ▎ "市场最高板的票现在什么状态？分歧还是一致？明天还能不能买？"

  现有接口的问题：
  - 没有高标股状态追踪
  - 没有分歧/一致判断
  - 没有断板风险评估

  应该扩展的能力：

  1. 高标股状态接口
     - 当前市场最高板是几板
     - 高标股今日表现（封单变化、成交额变化）
     - 高标股所属题材
     - 高标股明日风险评估（连续一字板后首次放量？）

  为什么重要：

  ▎ 高标股是市场情绪的风向标，高标断板往往意味着情绪拐点。

  多视角评审报告：短线交易能力扩展方案

  一、总体评价

  5 份功能文档整体质量很高，体现了对 A 股短线交易逻辑的深入理解。从代码架构角度看，三层分离（Vendor → Tool →
  Agent）的设计符合现有项目模式，实施路径清晰。但从实战交易者视角看，存在若干需要调整和补充的问题。以下分维度详述。
  
  ---
  二、逐文档评审

  📄 文档 01：连板梯队统计 + 情绪量化

  ✅ 做得好的：
  - 连板梯队分布、晋级率、封板质量等指标选取准确，覆盖了短线复盘的核心维度
  - 情绪周期的六阶段划分（冰点→修复→升温→高潮→退潮→分歧）在实战中确实有效
  - "超龙头视角简化"这部分特别好——实战中不需要复杂评分模型，3 个判断就够

  ⚠️ 需要修正的问题：

  1. 数据源假设有误。文档假设东财 RPT_LIMITUP_STOCK 接口有"连板天数"字段。但实际查看东财 datacenter API，涨停列表接口的字段命名和可用性需要实测验证。现有代码中 get_hot_stocks()  
  用的是同花顺的编辑精选数据，不是东财 raw 涨停列表。建议：先实测东财涨停列表接口的字段结构，确认 CONTINUOUS_LIMIT_NUM 字段确实存在且可用，再做方案。
  2. 封板率计算需要拆分"一字板"和"换手板"。文档提到"有效封板率 = 一字板数 + 早盘封板数(10:30前) / 涨停总数"，但实战中一字板和换手板的含义完全不同：
    - 一字板（开盘即封死）：说明利好强、筹码锁定好，但换手不充分，开板风险高
    - 换手板（盘中封板）：经过充分换手后封板，质量更稳定
    - 一字板不应简单计入"早盘封板"，应该分开统计
  3. 情绪周期判断缺少一个关键变量：北向资金流向。短线情绪和北向资金高度相关，尤其在转折点。当天北向大幅流出 + 高标分歧 = 确认退潮信号。现有代码已有
  get_northbound_flow()，应该纳入情绪综合评分。
  4. 闷杀率的阈值（>5%跌幅算闷杀）偏保守。A 股日内振幅经常 8-10%，昨日涨停股今日跌 5% 不算闷杀。实战中通常用：
    - 轻度闷杀：跌幅 > 3%（吃面但没大亏）
    - 重度闷杀：跌幅 > 7%（大幅亏钱）
    - 建议调整闷杀率的分级
  5. judge_strong_bullish_emotion 的阈值过于理想化。比如"昨日连板股平均溢价 >5%"——在正常市场中，连板股平均溢价通常在 2-5%，>5%
  只在极端强势行情才出现。如果按这个标准执行，几乎永远判不出"强看好"，失去了判断的意义。建议把阈值调低到实战常用值（如 >2% 即可）。
  6. 缺少"冰点确认"机制。文档里冰点的定义是"高标断板+梯队崩塌+闷杀率>30%"，但实战中冰点不是一天就能确认的。真正的冰点确认需要：
    - 连续 2 天高标断板
    - 连续 2 天梯队崩塌
    - 新题材没有持续性
    - 建议加入"冰点确认窗口"逻辑

  📄 文档 02：题材热度追踪

  ✅ 做得好的：
  - 题材阶段的六阶段划分（试探→发酵→主升→高潮→退潮→冰点）非常贴合实战
  - "看3日趋势而非2日"的思路很对
  - "结合龙头状态判断题材持续性"逻辑正确
  - 周五/节假日效应的特殊处理很实战

  ⚠️ 需要修正的问题：

  1. "涨停原因"的归因精度问题。东财 RPT_LIMITUP_STOCK 的涨停原因字段（通常叫 EXPLANATION 或 LIMIT_UP_REASON）经常是编辑人工标注的，且同一只票可能被归到多个原因。这导致：
    - 同一个概念可能被拆成多个细分方向
    - 一些票被标了错误的涨停原因
    - 建议：对涨停原因做归一化处理（比如"人工智能""AI""大模型"都归到"AI"）
  2. 题材热度的"持续天数"定义不够精确。文档定义"首次出现>=3只涨停的日期为启动日期"，但实战中：
    - 一个大题材（如 AI）可能有多个细分方向，启动日期不统一
    - 题材可能"断一天再续"，不是连续的
    - 建议用"近N日内出现涨停的天数"而非"连续天数"
  3. 题材热度评分的权重分配需要调整。文档中：
    - 涨停家数 40% + 最高连板 30% + 持续天数 20% + 龙头状态 10%
    - 实战中"龙头状态"的权重应该更高。一个有 5 板龙头的题材 vs 没有龙头的题材，即使涨停家数一样，前者的价值远高于后者
    - 建议：龙头状态权重提升到 20-25%，涨停家数降到 30%
  4. 缺少"题材辨识度"的量化方法。文档提到辨识度评估但没给出具体实现。实战中辨识度的核心指标是：
    - 是否有市场辨识度（社交媒体/财联社提及频率）
    - 是否有资金聚焦（封单集中度）
    - 是否有梯队支撑
    - 建议补充量化方法，可以用东财板块讨论热度或百度搜索指数作为代理变量

  📄 文档 03：首板筛选 + 二板预期

  ✅ 做得好的：
  - 封单强度的多维评估（峰值/稳定性/尾盘确认度/撤单次数）非常精细
  - 二板预期评分模型的因子选择合理
  - "超龙头视角说明"部分很关键——明确说明了本功能更适合首板选手而非龙头选手

  ⚠️ 需要修正的问题：

  1. 封单峰值数据的获取可行性存疑。文档假设能获取 seal_amount_peak（封单峰值），但东财 push2 的实时行情接口只返回当前封单金额，不返回日内峰值。要获取峰值需要：
    - 分时监控（盘中实时采集）
    - 或者用历史分时数据回推
    - 建议：承认这个数据限制，改用"当前封单/流通盘比"作为替代指标，或者从分时K线推算
  2. 撤单次数的数据获取不可行。东财没有公开的撤单次数 API。这个数据只能从 Level 2
  逐笔委托中获取，普通免费接口无法拿到。建议移除这个指标，或者改为"封单稳定性"（用分时封单金额的标准差估算）。
  3. "题材纯正度"的量化方法需要明确。文档提到"主营业务与题材直接相关→二板概率高"，但怎么量化？建议用简单方法：
    - 题材关键词在公司名称中的匹配度
    - 或者用同花顺/东财的题材纯度标注
    - 或者简化为：是否为题材内连板最高的票（是=纯正，否=不纯正）
  4. 二板预期评分的权重调整建议。当前权重：
    - 封单 25% + 量价 15% + 题材 20% + 情绪 15% + 时间 10% + 纯正度 10% + 股性 5%
    - 实战中"市场情绪"的权重应该更高。冰点期打首板的成功率远低于高潮期，情绪权重从 15% 提升到 20%，从封单里匀 5% 过来
  5. 流通市值阈值需要调整。文档用 20 亿和 50 亿作为分界线，但：
    - 当前 A 股市场 <20 亿流通市值的票流动性很差，不是好标的
    - 实战中短线活跃的首板票流通市值通常在 30-100 亿
    - 建议调整阈值：50 亿以下 +5 分，50-200 亿 +0 分，>200 亿 -5 分

  📄 文档 04：高标股状态监控

  ✅ 做得好的：
  - 分歧程度的多因子评估（换手率+开板次数+封单变化+分时形态+量比）很全面
  - 断板风险的高/中/低三级分类清晰
  - "卡位维度"的引入很关键——高标断板时如果有卡位股，情绪崩塌更快

  ⚠️ 需要修正的问题：

  1. 分歧度的评分模型过于复杂。文档定义了 5 个因子，但实战中：
    - 封单是否还在（是/否）→ 1 秒判断
    - 开板次数（0/1-2/3-5/>5）→ 1 秒判断
    - 这两个信息够用了。换手率、量比、分时形态在盘中实时获取困难，且对决策的边际贡献不大
    - 建议简化：核心分歧度 = 封单稳定性(50%) + 开板次数(50%)
  2. 一字板后开板的风险评估需要细化。文档说"连续一字板>=3天今日开板→高风险"，但实战中需要区分：
    - 一字板是缩量一字（无量涨停）→ 获利盘少，开板风险低
    - 一字板是放量一字（有换手）→ 获利盘多，开板风险高
    - 建议增加"一字板期间累计换手率"作为风险因子
  3. 缺少"分时封单变化趋势"的量化。文档提到"封单变化趋势"但没给出具体计算方法。实战中：
    - 封单持续增加 → 一致加强
    - 封单波动大 → 分歧
    - 封单骤降 → 即将炸板
    - 建议：用最近 30 分钟封单金额的标准差/均值作为"封单波动率"指标
  4. "板块效应"的获取方式需要明确。文档提到"高标股所属板块是否还强势"，但实现时需要：
    - 先获取高标股的概念板块
    - 再获取该板块的整体涨跌幅
    - 现有 get_industry_comparison() 是按行业分的，不是按概念板块分
    - 建议：用 get_concept_blocks() 获取概念板块行情，或新增概念板块涨跌幅接口

  📄 文档 05：龙头识别 + 卡位分析

  ✅ 做得好的：
  - 龙头评分的因子选择（高度+时间+封单+纯正度+认可度）准确
  - 卡位分析的三类划分（强卡位/弱卡位/补涨）逻辑清晰
  - "真假卡位判断"的条件设定（必须龙头分歧时才成立）很实战
  - 补涨龙空间评估（龙头高度-2）是经验法则，实用

  ⚠️ 需要修正的问题：

  1. 龙头评分的"流通市值扣分"逻辑需要调整。文档说"流通市值>100亿扣5分"，但：
    - 实际上很多市场龙头（如券商龙头、白酒龙头）流通市值都很大
    - 超短选手做龙头主要看连板高度和封板强度，流通市值是次要因素
    - 建议：改为"流通市值>200亿扣5分"，且仅在同板数比较时才扣分
  2. "首板时间最早"作为龙头判断因子的权重偏高。文档给了 20% 的权重，但：
    - 首板时间只在"同板数比较"时有意义
    - 不同板数的票比较首板时间没有意义（3 板的票肯定比 1 板的票涨停早）
    - 建议：首板时间只在同板数内部比较时使用，不参与跨板数评分
  3. 卡位强度评估的阈值需要调整。文档说"卡位票封单/龙头封单 >1.5 才算强卡位"，但：
    - 如果龙头封板状态不好（封单减少、开板），即使卡位票封单只有龙头的 1.2 倍，也可能是真卡位
    - 建议：增加"龙头封板状态"作为卡位判断的前提条件——龙头分歧时，卡位阈值降低
  4. 补涨龙的识别条件过于严格。文档要求"龙头连板数>=5"才考虑补涨龙，但实战中：
    - 4 板龙头就可能启动补涨龙
    - 尤其是大盘股龙头（如券商龙头），3-4 板时就可能有小盘补涨龙
    - 建议：降为"龙头连板数>=4 且封板状态为分歧"
  5. 缺少"补涨龙 vs 新龙头"的区分方法。文档提到"龙头断板后启动的可能是新龙头，不是补涨"，但没给出区分标准。实战中：
    - 补涨龙：龙头还在，低位票跟随上涨
    - 新龙头：龙头断板后，新的连板股出现
    - 建议增加区分逻辑

  三、架构层面的问题

  1. Agent 接入位置不合理

  文档计划将所有新工具接入 market_analyst 和 hot_money_tracker，但：
  - market_analyst 当前只用 get_stock_data + get_indicators（技术面分析）
  - hot_money_tracker 当前有 14 个工具（资金面+消息面）
  - 连板梯队、情绪量化、高标状态、龙头识别这些属于短线博弈分析，既不是纯技术面也不是纯资金面

  建议：创建一个独立的 short_term_analyst（短线博弈分析师），专门处理这 5 个新功能。 这样：
  - 职责更清晰
  - 工具集更聚焦
  - Agent 的 system prompt 可以针对性设计
  - 不会污染现有 Agent 的工具列表

  2. 缺少 State 字段

  AgentState 当前没有 emotion_report 或 short_term_report 字段。5 个新功能的结果需要一个地方存储。

  建议：在 agent_states.py 中新增 short_term_report 字段，由新的 short_term_analyst 填充。

  3. Prefetch 层需要更新

  新增 5 个 tool 后，prefetch.py 中的预取任务列表需要同步更新。特别是连板梯队和情绪量化，应该是全市场级别的预取（不依赖个股 ticker）。

  4. 测试数据的特殊性

  短线交易功能的测试数据有特殊性：
  - 涨停/跌停数据只在交易日存在
  - 非交易日查询应该返回空数据或上一交易日数据
  - 周末/节假日的数据需要特殊处理

  建议：测试用例中增加"非交易日查询"和"数据为空"的边界测试。

  四、缺失的关键维度

  从资深交易者视角看，文档覆盖了短线交易的大部分维度，但遗漏了几个实战中非常重要的指标：

  1. 市场涨跌家数比（AD 比）

  这是最基础的市场宽度指标。涨停家数只看最强的票，但涨跌家数比反映的是全市场的真实温度。
  - 涨 > 跌 3:1 → 强势
  - 涨 ≈ 跌 1:1 → 分歧
  - 跌 > 涨 3:1 → 弱势
  - 建议新增 get_market_breadth() 接口

  2. 昨日涨停股的"竞价表现"

  盘前最重要的情绪指标之一是昨日涨停股的集合竞价表现：
  - 昨日涨停股平均高开幅度
  - 昨日涨停股竞价量比
  - 这决定了当日情绪预期
  - 但这个数据需要盘中实时采集，目前免费接口无法获取竞价数据。建议在文档中说明这个限制。

  3. 市场总成交额

  短线情绪和市场总成交额高度相关：
  - 成交额持续放大 → 市场活跃 → 赚钱效应好
  - 成交额持续萎缩 → 市场冷淡 → 赚钱效应差
  - 建议在情绪量化中加入"市场总成交额"和"5日平均成交额对比"

  4. 融券余额变化

  融券余额增加 = 看空力量增强，融券余额减少 = 空头回补。现有 get_margin_trading() 已有融资融券数据，应该纳入情绪综合评分。
